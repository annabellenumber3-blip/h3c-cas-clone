Configure CIC backup

Perform this task to configure the path for saving CIC backup files, enable or disable automatic backup, and set automatic backup parameters.

Procedure

  1. From the left navigation pane, select System > CIC Backup.

  1. Select the backup destination. Options include Local Host and Remote Server.

  • If you select Local Host, click  to select a backup location, or enter a backup location.

  • If you select Remote Server, specify an IP address, username, password, and backup location, select a connection type, and specify a service port, and then click Connectivity Test.

  1. To enable scheduled backup, click Scheduled Backup, and select the frequency, date, time, and backups to save as needed.

  1. Click Save.

Parameters

  • Destination: Select a backup destination. Options include Local Host and Remote Server.

  • Location: If you select Local Host as the backup destination, this parameter indicates the directory on the local host to save the CIC backup files, /vms/images, for example. As a best practice, do not specify the root directory as the location. If you select Remote Server as the backup destination, this parameter indicates the directory on the remote server to save the CIC backup files.

  • IP Address: Specify the IP address of the remote server. This field is required if you select Remote Server as the backup destination.

  • Username: Specify an account for the remote server. This account must have read and writes rights. If you select FTP as the connection type, this account must be an account created on the remote FTP server. If you select SCP, this account must be an account created on the OS of the remote server. This field is required if you select Remote Server as the backup destination.

  • Password: Password used for connecting to the remote server. This field is required if you select Remote Server as the backup destination.

  • Connection Type: Specify a connection type for connecting to the remote server. Options include FTP and SCP. This field is required if you select Remote Server as the backup destination.

  • Server Port: Specify a port for accessing the remote server. This field is required if you select Remote Server as the backup destination.

  • Frequency: Specify the frequency at which the backup policy is executed. Options include Monthly, Weekly, and Daily.

  • Monthly: Configuration backup is executed at the specified time every month.

  • Weekly: Configuration backup is executed at the specified time every week.

  • Daily: Configuration backup is executed at the specified time every day.

  • Date: Specify the date to execute the backup policy. This parameter is required if you select Monthly as the frequency.

  • Day: Specify the day to execute the backup policy. This field is required if you select Weekly as the frequency.

  • Time: Specify the time at which the backup policy is executed.

  • Backups to Save: Number of backup files to save. The number of backup files to save is not limited if you leave this field empty.
