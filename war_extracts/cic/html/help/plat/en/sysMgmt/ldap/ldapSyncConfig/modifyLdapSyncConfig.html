Modify an LDAP synchronization policy

Perform this task to modify an LDAP synchronization policy.

Procedure

  1. From the left navigation pane, select System > LDAP User Sync > Sync Policies.

  1. Click the  icon in the Action column for the target LDAP synchronization policy.

  1. In the dialog box that opens, select an LDAP server and sub base DN, and specify a filtering criterion as needed, and then click Next.

  1. Select Synchronize Groups by OU or Not Synchronize Groups from the Synchronize User Group list. If you select Synchronize Groups by OU, select a father user group. If you select Not Synchronize Groups, select a user group. Select a configuration state and then click Next.

  1. Select synchronize options as needed, including automatic synchronization, on-demand synchronization, synchronize new users, synchronize existing users, synchronize current users, and log off non-existent users, and then click Next.

  1. Select an attribute corresponding to username, identity number, contact, phone number, and email address, respectively.

  1. Click Finish.

Parameters

  • Sync Policy Name: Name of the LDAP synchronization policy.

  • LDAP Server: Select an LDAP server used by the LDAP synchronization policy. Options include all LDAP servers that the current operator can display.

  • Base DN: Absolute path of the directory that stores user data on the LDAP server. This field automatically displays the corresponding base DN when you select an LDAP server.

  • Sub-Base DN: Select a sub-base DN for LDAP search. Options include all sub base DNs of the base DN.

  • Filter Criteria: Enter the criteria for filtering users in the base DN corresponding to the LDAP server.

  • Organization: Select an organization.

  • Synchronize User Group: Select whether to synchronize user groups. Options include Not Synchronize Groups and Synchronize Groups by OU.

  • Synchronize Groups by OU: The system synchronizes OUs in the specified sub base DN as user groups to an organization, and forms user group relations based on the OU mappings in LDAP.

  • Not Synchronize Groups: The system does not synchronize OUs as user groups to an organization. All users of the sub base DN are synchronized to the user group specified in this policy.

  • Father User Group: Select a father user group or leave this field empty as needed. If you leave this field empty, the sub base DN will be used as the father user group, and the users and OUs in this sub base DN will be synchronized to the father user group specified in the policy. If you select a father user group, the users and OUs in this sub base DN will be synchronized to the father user group specified in the policy.

  • Configuration State: Select a state for the LDAP synchronization policy. Options include Valid and Invalid.

  • Policy

  • Auto Sync: Enable or disable automatic synchronization as needed. If you enable this option, the policy automatically synchronizes users at 00:00 every day.

  • On-Demand Sync: Enable or disable on-demand synchronization. If you enable this option, the system automatically adds a user when the user logs in to CIC SSV.

  • Synchronize New Users: If you select this option, the policy only synchronizes new users.

  • Synchronize Existing Users: If you select this option, the policy only synchronizes users existing in the system.

  • Synchronize Current Users: If you select this option, the policy only synchronizes users of the current sub base DN. If you do not select this option, the policy synchronizes users of the current sub base DN and all its sub base DNs.

  • Log Off Non-Existent Users: If you select this option, the policy deletes users that have been synchronized to CIC through this policy but not existing on the LDAP server. If you do not select this option, the state of the corresponding users becomes invalid.

  • User Info

  • Login Name: This field is unconfigurable.

  • Username: Select an attribute name corresponding to the username.

  • Identity No: Select an attribute name corresponding to the identity number.

  • Contact: Select an attribute name corresponding to the contact address.

  • Phone Number: Select an attribute name corresponding to the phone number.

  • Email: Select an attribute name corresponding to the email address.
