Display LDAP synchronization policy details

Perform this task to display detailed information about an LDAP synchronization policy.

Procedure

  1. From the left navigation pane, select System > LDAP User Sync > Sync Policies.

  1. Click the  icon in the Action column for the target LDAP synchronization policy.

Parameters

Basic configuration

  • Sync Policy Name: Name of the LDAP synchronization policy.

  • LDAP Server: Name of the LDAP server used by the LDAP synchronization policy.

  • Base DN: Absolute path of the directory that stores user data on the LDAP server.

  • Sub-Base DN: Sub-base DN for LDAP search.

  • Filter Criteria: Criteria for filtering users in the base DN corresponding to the LDAP server.

  • Organization: Name of the organization to which users are synchronized through the LDAP synchronization policy.

  • Synchronize User Group: Whether to synchronize user groups.

  • Synchronize Groups by OU: The system synchronizes OUs in the specified sub base DN as user groups to an organization, and forms user group relations based on the OU mappings in LDAP.

  • Not Synchronize Groups: The system does not synchronize OUs as user groups to an organization. All users of the sub base DN are synchronized to the user group specified in this policy.

  • Father User Group: Father user group of the user group to which the OUs belong. If this field is empty, the sub base DN will be used as the father user group, and the users and OUs in this sub base DN will be synchronized to the father user group specified in the policy. If it is not empty, the users and OUs in this sub base DN will be synchronized to this father user group.

  • User Group: Name of the user group to which the policy synchronizes the user.

  • Configuration State: State of the LDAP synchronization policy.

Sync Options

  • Auto Sync: The policy automatically synchronizes users at 00:00 every day.

  • On-Demand Sync: The system automatically adds a user when the user logs in to CAS SSV.

  • Synchronize New Users: The policy only synchronizes new users.

  • Synchronize Existing Users: The policy only synchronizes users existing in CIC.

  • Synchronize Current Users: The policy only synchronizes users of the current sub base DN. If you do not select this option, the policy synchronizes users of the current sub base DN and all its sub base DNs.

  • Log Off Non-Existent Users: The policy deletes users that have been synchronized to CIC through this policy but not existing on the LDAP server. If you do not select this option, the state of the corresponding users becomes invalid.

  • User Info

  • Login Name: This field is unconfigurable.

  • Username: Select an attribute name corresponding to the username.

  • Identity No: Select an attribute name corresponding to the identity number.

  • Contact: Select an attribute name corresponding to the contact address.

  • Phone Number: Select an attribute name corresponding to the phone number.

  • Email: Select an attribute name corresponding to the email address.
