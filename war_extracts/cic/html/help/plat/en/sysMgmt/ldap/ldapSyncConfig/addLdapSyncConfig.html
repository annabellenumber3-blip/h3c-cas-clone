Add an LDAP synchronization policy

Perform this task to add an LDAP synchronization policy.

Procedure

  1. From the left navigation pane, select System > LDAP User Sync > Sync Policies.

  1. Click Add.

  1. In the dialog box that opens, specify a policy name and filtering criterion, select an LDAP server and sub base DN, and then click Next.

  1. Select an organization and user group type. Select Synchronize Groups by OU or Not Synchronize Groups from the Synchronize User Group list. If you select Synchronize Groups by OU, select a father user group. If you select Not Synchronize Groups, select a user group. Select a configuration state and then click Next.

  1. Configure the following parameters for the synchronization policy as needed, and then click Next: Auto Sync, On-Demand Sync, Synchronize New Users, Synchronize Existing Users, Synchronize Current Users, and Log Off Non-Existent Users.

  1. Select the user attributes corresponding to the username, identity number, contact address, phone number, and email.

  1. Click Finish.

Parameters

  • Sync Policy Name: Enter the name of the LDAP synchronization policy.

  • LDAP Server: Select the LDAP server used by the LDAP synchronization policy.

  • Base DN: Absolute path of the directory that stores user data on the LDAP server. This field automatically displays the corresponding DN when you select an LDAP server.

  • Sub-Base DN: Select a sub-base DN for LDAP search. Options include all sub-base DNs of the base DN.

  • Filter Criteria: Enter the criteria for filtering users in the base DN corresponding to the LDAP server.

  • Organization: Select the organization to which users are synchronized through the LDAP synchronization policy.

  • Synchronize User Group: Select a synchronization type. Options include Not Synchronize Groups and Synchronize Groups by OU.

  • Synchronize Groups by OU: The system synchronizes OUs in the specified sub base DN as user groups to an organization, and forms user group relations based on the OU mappings in LDAP.

  • Not Synchronize Groups: The system does not synchronize OUs as user groups to an organization. All users of the sub base DN are synchronized to the user group specified in this policy.

  • Father User Group: Select a father user group or leave this field empty as needed. If you leave this field empty, the sub base DN will be used as the father user group, and the users and OUs in this sub base DN will be synchronized to the father user group specified in the policy. If you select a father user group, the users and OUs in this sub base DN will be synchronized to the father user group specified in the policy.

  • User Group: Select the user group to which the policy synchronizes the user.

  • Configuration State: Select the state of the LDAP synchronization policy. Options include Valid and Invalid.

  • Policy

  • Auto Sync: The policy automatically synchronizes users at 00:00 every day.

  • On-Demand Sync: The system automatically adds a user when the user logs in to CAS SSV.

  • Synchronize New Users: The policy only synchronizes new users.

  • Synchronize Existing Users: The policy only synchronizes users existing in CIC.

  • Synchronize Current Users: The policy only synchronizes users of the current sub base DN. If you do not select this option, the policy synchronizes users of the current sub base DN and all its sub base DNs.

  • Log Off Non-Existent Users: The policy deletes users that have been synchronized to CIC through this policy but not existing on the LDAP server. If you do not select this option, the state of the corresponding users becomes invalid.

  • User Info

  • Login Name: This field is unconfigurable.

  • Username: Select an attribute name corresponding to the username.

  • Identity No: Select an attribute name corresponding to the identity number.

  • Contact: Select an attribute name corresponding to the contact address.

  • Phone Number: Select an attribute name corresponding to the phone number.

  • Email: Select an attribute name corresponding to the email address.
