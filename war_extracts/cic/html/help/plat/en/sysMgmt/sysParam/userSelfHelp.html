Configure self-service system parameters

Perform this task to set self-service system parameters.

Restrictions and guidelines

Use a client tool such as SSH to put the icon files to the /var/lib/tomcat8/webapps/ssv/images directory, and set the path to ./images/image name. File names do not support Chinese characters. For best results, use the recommended image size.

Procedure

  1. From the left navigation pane, select System > Parameters.

  1. Click the User Self-Service System Params tab.

  1. Specify the system name, system icon path, system icon height and width, copyright information, vendor logo path, and vendor logo height and width.

  1. Enable VNC and snapshot, cloud backup, cloud message, cloud disk, support ticket, and cloud alarm management as needed.

  1. Click Save.

Parameters

Self-Service System Params

  • System Name: Set the system name displayed on the login page.

  • System Icon: Set the path of the system icon displayed on the login page. For best results, use a 163 × 103 pixel image size.

  • System Icon Height: Set the height of the system icon displayed on the login page.

  • System Icon Width: Set the width of the system icon displayed on the login page.

  • Copyright Info: Set the copyright information.

  • Vendor Logo: Specify the path of the vendor logo. For best results, use a 69 × 18 pixel image size.

  • Vendor Logo Height: Set the height of the vendor logo.

  • Vendor Logo: Set the width of the vendor logo.

  • Snapshot Mgmt: Enable or disable snapshot management. The snapshot of a VM is the copy of the disk file, memory, and configuration file at a specific time. Operators can restore the VM to the state it was in at the time you took the parent snapshot.

  • VNC: Enable or disable VNC remote connection. If you enable VNC, users can log in to operate cloud hosts through VNC.

  • Cloud Backup Mgmt: Enable or disable cloud backup management. If you enable cloud backup management, users can log in to SSV to restore cloud hosts through cloud backup.

  • Cloud Message Mgmt: Enable or disable cloud message management. If you enable cloud message management, users can log in to SSV to view messages sent by administrators.

  • Cloud Disk Mgmt: Enable or disable cloud disk management. If you enable cloud disk management, users can log in to SSV to request for disks and attach or detach disks for cloud hosts.

  • Support Ticket Mgmt: Enable or disable support ticket management. If you enable support ticket management, users can log in to SSV to communicate with administrators to resolve problems.

  • Cloud Alarm Mgmt: Enable or disable cloud alarm management. If you enable cloud alarm management, users can log in to SSV to view cloud alarms.

  • Online Services: Enable or disable online services. If you enable online services, users can log in to SSV to get online services.
