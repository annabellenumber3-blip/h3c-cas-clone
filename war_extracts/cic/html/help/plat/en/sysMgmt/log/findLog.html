Filter operation logs

Perform this task to filter operation logs.

Restrictions and guidelines

  • Operation logs with a login name of $SYSTEM are automatically recorded by the system after scheduled tasks are completed. For example, when the system automatically backs up VMs, the system automatically records operation logs of VM backup, with a login name of $SYSTEM and username of $SYSTEM.

  • Operation logs with a login name of $admin@VMC-RS are automatically recorded by the system when a third-party system calls the REST interface of this system. For example, when a tenant starts a VM through CAS SSV, the system automatically records operation logs for starting the VM, with a login name of $admin@VMC-RS and username of @VMC-RS.

Procedure

  1. From the left navigation pane, select System > Operation Logs.

  1. Click Filter.

  1. In the dialog box that opens, set the filtering criteria as required.

  1. Click OK. All operators matching the criteria will be displayed on the list.

Parameters

  • Login Name: Enter the login name used by the operator to log in to CIC.

  • Operator: Specify a name for the operator.

  • Start From: Specify the start time. Click on this field, or click to select a start time from the calendar dialog box that opens.

  • End On: Specify the end time Click on this field, or click to select a start time from the calendar dialog box that opens.

  • Login Address: Enter an IP address used by the operator to log in to CIC.

  • Action: Select the actions that the operator has taken. Options include operator action, user actions, system configuration, data center action, host pool action, host action, storage action, network action, cluster action, VM action, backup action, alarm management action, service workflow action, organization action, virtual storage action, virtual desktop pool action, cloud disk action, and external cloud resource action.

  • Result: Select an operation result. Options include All, Succeeded, Failed, and Partially Succeeded.
