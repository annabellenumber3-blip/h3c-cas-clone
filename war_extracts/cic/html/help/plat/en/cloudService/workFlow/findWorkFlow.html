Perform advanced filtering for cloud host workflows

Perform this task to manage the workflows submitted by terminal users for requesting or canceling cloud hosts or extending the time to use the requested cloud hosts through CAS SSV.

Procedure

  1. From the left navigation pane, select Workflows > Cloud Host Tasks.

  1. Click Advanced Filter.

  1. In the dialog box that opens, set the filtering criteria as required.

  1. To reset the filtering criteria, click Reset.

  1. Click OK.

All cloud host workflows matching the criteria will be displayed on the list.

Parameters

  • Username: Username used by the user to request the cloud host.

  • Template Name: Name of the VM template used for requesting the cloud host.

  • Status: Status of the cloud host workflow, including To Be Processed, Approved, Denied, Canceled, Deployment Success, Deployment Failure, Deletion Success, Deletion Failure, and All.

  • Type: Type of the cloud host workflow, including Application, Cancellation, and Extension.

  • Creation Time (From): Start time for filtering cloud host workflows by creation time. Click on this field, or click the  icon to select a start time from the calendar dialog box that opens.

  • Creation Time (To): End time for filtering cloud host workflows by creation time. Click on this field, or click the  icon to select an end time from the calendar dialog box that opens.

  • Process Time (From): Start time for filtering cloud host workflows by processing time. Click on this field, or click the  icon to select a start time from the calendar dialog box that opens.

  • Process Time (To): End time for filtering cloud host workflows by processing time. Click on this field, or click the  icon to select an end time from the calendar dialog box that opens.
