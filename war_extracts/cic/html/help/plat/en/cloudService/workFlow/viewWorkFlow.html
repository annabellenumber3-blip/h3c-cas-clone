Display cloud host workflows

Perform this task to display workflows submitted by terminal users for requesting, canceling, or extending the time to use the requested cloud hosts. By default, the cloud host workflow list displays only cloud host workflows in To Be Processed state. Organization administrators can view cloud host workflows in other states.

Procedure

From the left navigation pane, select Workflows > Cloud Host Tasks.

Parameters

  • Name: Name of the cloud host requested or canceled by the user.

  • Alias: Alias of the cloud host requested or canceled by the user.

  • Template Name: Name of the VM template used for requesting the cloud host.

  • Status: State of the cloud host workflow.

  • Type: Type of the cloud host workflow.

  • Username: Username used by the user to request the cloud host.

  • Purpose: Reason why the user has requested the cloud host.

  • Treatment: Treatment on the workflow by the organization administrator.

  • Requested At: Time when the user submitted the workflow.

  • Processed At: Time when the organization administrator processed the workflow.

  • Computer Name: Computer name of the cloud host that has been deployed successfully.

  • Domain/Workgroup: Name of the domain/workgroup to which the cloud host that has been deployed successfully belongs.

  • Login Name: Username for the cloud host that has been deployed successfully.

  • Belong To: The cloud host that has been deployed successfully belongs to a domain or work group.

  • CPUs: Number of CPU cores on the cloud host that has been deployed successfully.

  • Memory: Memory size of the cloud host that has been deployed successfully.

  • Action: Actions that operators can perform on the cloud host workflow.
