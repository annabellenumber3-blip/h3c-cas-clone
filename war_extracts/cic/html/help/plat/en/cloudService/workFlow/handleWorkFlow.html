Deploy VMs requested in a cloud host workflow

Perform this task to deploy VMs requested in a cloud host workflow.

Procedure

  1. From the left navigation pane, select Workflows > Cloud Host Tasks.

  1. Click the  icon in the Action column for the target cloud host workflow.

  1. In the dialog box that opens, enter an alias and comment for the VM, and select a resource pool and virtual desktop pool.

  • If the user is a CVM organization user and the VM uses a predefined template, enable Fast Deployment and Configure OS by clicking on the toggle button, and then click Next.

  • If the user is a CVM organization user and the VM is a user-defined VM, select Use Template. Then determine whether to enable Fast Deployment by clicking on the toggle button, select a VM template, and then click Next.

  1. Set the number of CPUs and memory size for the VM. If the current organization does not allow VM modification, these parameters are unconfigurable.

  1. Set the cloud disk size, select an expiry date, and destination storage pool.

  1. Select a virtual switch and port profile in the advanced settings.

  1. Click Finish if you did not select Configure OS when you deploy VMs through predefined templates.

  1. Click Next to configure the operating system for the VM if you selected Configure OS when you deploy VMs through predefined templates.

  1. Enter a computer name, and select an IP address allocation mode. If you select DHCP, click Next. If you select Manual, enter an IP address, subnet mask, default gateway, primary DNS server, and secondary DNS server, and then click Next.

  1. Configure login information:

  • To deploy VMs that use templates that use a Windows system and add the deployed VMs to a domain or work group, enter the domain or work group name, login name, password, confirm password, and activation serial number, select a local group, and then click Finish.

  • To deploy a VM that uses a Linux template, enter an account, login password, and confirm password, and then click Finish.

Parameters

  • Alias: Enter the alias of the VM.

  • Comment: Enter the comment on VM deployment.

  • Resource Pool Name: Select a cluster to which the VM is to be deployed.

  • Virtual Desktop Pool: Select a virtual desktop pool for the VM to be deployed.

  • Fast Deployment: Select this option to create the VM based on the base image file of the VM. Fast deployment saves storage spaces of servers.

  • CPU: Enter the number of CPUs of the VM.

  • CPU Cores: Enter the number of CPU cores of the VM.

  • Memory: Enter the memory size of the VM.

  • Cloud Hard Disk: Enter the cloud hard disk size of the VM. This parameter is available when the VM to be deployed uses the pre-defined template and requires a cloud hard disk.

  • Expiry Date: Select a date on which the VM will expire.

  • Port Profile: Select a port profile for the VM.

  • Storage Pool: Select a storage pool for the VM.

  • vSwitch: Select a vSwitch for the VM.

  • Configure OS: Select this option to configure the operating system for the VM.

  • Initialization Type: Select an initialization type to configure initial OS and user account settings. This operation does not clear SID information for the OS. Fast initialization requires installation of CAStools. This type is applicable to Windows and Linux operating systems that support CAStools.

  • Computer Name: Enter the computer name in the operating system.

  • IP Assignment: Select an IP assignment mode. Options include DHCP and Manual.

  • IP Address: Enter the IP address of the VM.

  • Subnet Mask: Enter the subnet mask of the VM.

  • Default Gateway: Specify a default gateway for the VM.

  • Primary DNS Server: Specify a primary DNS server for the VM.

  • Secondary DNS Server: Specify a secondary DNS server for the VM.

  • Belong To: Configure the VM to belong to a domain or workgroup. This parameter is required only when the VM uses a Windows template.

  • Domain/Workgroup: Specify a domain or workgroup for the VM. This parameter is required only when the VM uses a Windows template.

  • Login Account: Enter the account for logging in to the OS of the VM.

  • Password: Enter the password of the login account.

  • Confirm: Enter the confirm password. It must be the same as the login password.

  • Local Group: Select the local group to which the login account belongs. Options include Administrators, Power Users, and Users. This parameter is required only when the VM uses a Windows template.

  • Activate Serial Number: Enter the activate serial number for the OS of the VM. This parameter is required only when configuring VM OS and full initialization are selected. If you do select enter an activate serial number, you must enter a correct activate serial number when the VM starts up.

  • Additional Field: Enter additional field information. This parameter is required only when additional fields are configured.
