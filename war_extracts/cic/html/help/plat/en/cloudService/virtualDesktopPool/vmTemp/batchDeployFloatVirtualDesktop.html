Bulk deploy floating virtual desktops

Perform this task to bulk deploy virtual desktops by using a VM template in a floating virtual desktop pool.

Restrictions and guidelines

You cannot manually assign floating virtual desktops to users. When a user in the user group assigned a floating desktop pool logs in to SSV, the system automatically assigns a random virtual desktop to the user. When the user logs out of SSV, the system reclaims the virtual desktop used by the user.

Procedure

  1. From the left navigation pane, select VDC > Virtual Desktop Pools > virtual desktop pool name.

  1. Click the VM Templates tab.

  1. Click the  icon in the Action column for a VM template.

  1. Enter the start IP address, end IP address, subnet mask, default gateway, primary DNS server address, backup DNS server address, and extension command, and then click OK.

Parameters

  • Start IP: Enter the start IP address of the IP address range assigned to the floating desktop pool. The address must be in dotted-decimal format.

  • End IP: Enter the end IP address of the IP address range assigned to the floating desktop pool. The address must be in dotted-decimal format. Make sure the IP address range contains enough IP addresses for the virtual desktops in the floating desktop pool.

  • Subnet Mask: Enter a subnet mask in dotted-decimal format.

  • Default Gateway: Enter the default gateway address in dotted-decimal format.

  • Primary DNS Server: Enter the primary DNS server address in dotted-decimal format.

  • Backup DNS Server: Enter the backup DNS server address in dotted-decimal format.

  • Extension Command: Enter a script or command for the virtual desktop to automatically execute in a shell or console after its OS starts.
