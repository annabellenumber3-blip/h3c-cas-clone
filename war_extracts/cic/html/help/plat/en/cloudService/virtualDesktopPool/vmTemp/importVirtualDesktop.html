Import virtual desktop configurations

Perform this task to import configurations of multiple virtual desktops from a file during bulk virtual desktop deployment.

Restrictions and guidelines

  • To avoid import failure, make sure the columns you specify match the columns in the file.

  • If an imported virtual desktop name or alias already exists in CIC, the virtual desktop cannot be deployed.

  • To avoid import failure, make sure the delimiters you select match the delimiters in the file.

Procedure

  1. From the left navigation pane, select VDC > Virtual Desktop Pools > virtual desktop pool name.

  1. Click the VM Templates tab.

  1. Click the  icon in the Action column for a VM template.

  1. Select whether to enable the protection mode and quick deployment.

  1. Click Import from File.

  1. In the dialog box that opens, click Browse to select a file. You can click Preview to preview the columns in the file. Select a delimiter, specify the start row, select an initialization type, and click Next.

  1. Specify the columns that indicate the alias, description, number of CPUs, number of CPU cores, memory size, and memory unit, respectively.

  1. Select the assignment mode.

  • If you select the Not Assign option, click Next.

  • If you select the Assign to User or Assign to User Group option, specify the column that indicates users or user groups, and click Next.

  1. Specify the column that indicates the computer name and select the IP address assignment mode.

  • If you select the Manual option, specify the columns that indicate the IP address, subnet mask, default gateway, primary DNS server address, and secondary DNS server address, respectively. Select the IP Binding option as needed, and click Next.

  • If you select the DHCP option, click Next.

  1. Select whether the virtual desktops belong to a domain or workgroup, and specify the columns that indicate the domain/workgroup, login account, password, and activation serial number, respectively. Select a local group.

  1. Click Finish.

Parameters

  • Select File: Select a file that contains virtual desktop configurations. You can create a file or click the Template File link to customize the template file. The system supports TXT or SCV files that use commas, semicolons, the Tab keys, or spaces to separate columns.

  • Separator: Select a column separator. Options include comma, semicolon, the tab key, and space.

  • Start Row: Specify the start row of configuration information.

  • Initialization Mode: Select an initialization mode. Options include Quick Initialization and Full Initialization.

  • Quick Initialization: When deploying a VM, the system uses CAStools to initialize the operating system parameters and user account information of the VM and does not clear the SID information. The OS of the VM template must be installed with CAStools. Quick initialization applies to VM templates with Windows or Linux OSs.

  • Full Initialization: When deploying a VM, the system uses Sysprep to initialize the operating system parameters and user account information of the VM and clears the SID information. The OS of the VM template must be installed with Sysprep. Full initialization applies to VM templates with Windows 7 and Windows XP OSs.

  • Alias: Select the column that indicates the virtual desktop alias.

  • Description: Select the column that indicates the description.

  • CPUs: Select the column that indicates the number of CPUs. You can set this parameter only when the organization that the virtual desktop pool belongs to allows modification of VM configuration.

  • CPU Cores: Select the column that indicates the number of CPU cores. You can set this parameter only when the organization that the virtual desktop pool belongs to allows modification of VM configuration.

  • Memory Size: Select the column that indicates the memory size. The unit is MB. You can set this parameter only when the organization that the virtual desktop pool belongs to allows modification of VM configuration.

  • Assignment Mode: Select the objects you want to assign the virtual desktops to.

  • User/User Group: Select the column that indicates the users or user groups. You must set this parameter if you select the Assign to User or Assign to User Group option.

  • Computer Name: Select the column that indicates the computer name.

  • IP Assignment: Select an IP address assignment mode. Options include DHCP and Manual.

  • IP Address: Select the column that indicates the IP address. This parameter is required if you select the Manual option.

  • Subnet Mask: Select the column that indicates the subnet mask. This parameter is required if you select the Manual option.

  • Default Gateway: Select the column that indicates the default gateway.

  • Primary DNS Server: Select the column that indicates the primary DNS server address.

  • Secondary DNS Server: Select the column that indicates the secondary DNS server address.

  • IP Binding: Select whether to bind the IP addresses to the virtual desktops by clicking the toggle button. This parameter is configurable if you select the Manual option.

  • Belong To: Select whether the virtual desktops belong to a domain or workgroup. This parameter is configurable only when you use a Windows VM template.

  • Domain/Workgroup: Select the column that indicates the domain or workgroup. This parameter is configurable only when you use a Windows VM template.

  • Login Account: Select the column that indicates the login account.

  • Password: Select the column that indicates the password.

  • Local Group: Select a local group for the login accounts. Options include Administrators, Power Users, and Users. This parameter is configurable only when you use a Windows VM template.

  • Activation Serial Number: Select the column that indicates the activation serial number. This parameter is configurable only when you use a Windows VM template and select full initialization. If you do not set this parameter, the virtual desktop prompts for an activation serial number that matches the OS version after startup.
