Deploy a virtual desktop

Perform this task to deploy a new virtual desktop in a virtual desktop pool by using a VM template.

Restrictions and guidelines

  • When you use a VM template to deploy virtual desktops, make sure the VM template meets the following requirements: has an OS, installed with CAStools and Sysprep, uses a NIC in DHCP mode, enabled with remote desktop, and has only the system default user in the OS.

  • When you use a Windows Server VM template to deploy a virtual desktop, make sure the password you set meets the requirements in the password policy defined in the VM template, including the minimum password length, password complexity, and password history requirements.

Procedure

  1. From the left navigation pane, select VDC > Virtual Desktop Pools > virtual desktop pool name.

  1. Click the VM Templates tab.

  1. Click the  icon in the Action column for a VM template.

  1. Enter the VM alias and description, select the initialization type, set whether to enable the protection mode and quick deployment, and then click Next.

  • If the organization that the virtual desktop pool belongs to allows modification of VM configuration, configure CPU and memory settings as described in step 5.

  • If the organization that the virtual desktop pool belongs to does not allow modification of VM configuration, go to step 6.

  1. Specify the number of CPUs, the number of CPU cores, and the memory size.

  1. Select the assignment mode.

  • Not Assign: Does not assign the virtual desktop to any user or use group.

  • Assign to User: Assigns the virtual desktop to users. If you select this option, click the  icon. In the dialog box that opens, select users from the Users Not Assigned the VM list and click Add. After you select all target users, click OK.

  • Assign to User Group: Assigns the virtual desktop to user groups. If you select this option, click the  icon. In the dialog box that opens, select user groups and click OK.

  1. In advanced settings, select a virtual switch, select a port profile, and click Next.

  1. Enter the computer name and select an IP address assignment mode.

  • If you select the DHCP option, click Next.

  • If you select the Manual option, enter the IP address, subnet mask, default gateway, primary DNS server address, and secondary DNS server address, select the IP Binding option as needed, and then click Next.

  1. Configure login information.

  • If you use a Windows VM template, specify a domain or workgroup, enter the login account and password, enter the activation serial number, and select a local group.

  • If you use a Linux VM template, enter the login account and password.

  1. Click Finish.

Parameters

  • Protection Mode: Set whether to enable the protection mode. In this mode, the incremental files of VM disks cannot be modified. If a user modifies the disk data of a VM, the data is automatically initialized when the VM restarts. If you enable the protection mode, quick deployment is also enabled.

  • Quick Deployment: Enable VM deployment based on the image files of the VM template. To shorten the VM creation time and save server storage space, select this option.

  • CPUs: Specify the number of CPUs and the number of CPU cores. You can set this parameter only when the organization that the virtual desktop pool belongs to allows modification of VM configuration.

  • Memory: Specify the memory size. You can set this parameter only when the organization that the virtual desktop pool belongs to allows modification of VM configuration.

  • Assignment Mode: Select the objects you want to assign the virtual desktop to.

  • Not Assign: Does not assign the virtual desktop to any user or use group.

  • Assign to User: Assigns the virtual desktop to users.

  • Assign to User Group: Assigns the virtual desktop to user groups.

  • User: Select the users to be assigned the virtual desktop. This parameter is required if you select the Assign to User option. Click the input box or the icon to the right of this parameter to select one or multiple users.

  • User Group: Select the user groups to be assigned the virtual desktop. This parameter is required if you select the Assign to User Group option. Click the input box or the icon to the right of this parameter to select one or multiple user groups.

  • Profile: Select a profile.

  • vSwitch: Select a vSwitch for the virtual desktop.

  • Initialization Type: Select an initialization type.

  • Fast: Uses CAStools to set OS parameters and user account information for the virtual desktop and does not clear SID information of the OS. Fast initialization requires CAStools to be installed on the VM template. This mode applies to a Windows or Linux OS that supports CAStools.

  • Full: Uses Sysprep to set OS parameters and user account information for the virtual desktop and initialize SID information of the OS. Full initialization requires Sysprep to be installed on the VM template. This mode applies to Windows 7 and Windows XP.

  • Computer Name: Enter a computer name.

  • IP Assignment: Select an IP address assignment mode.

  • IP Address: Enter an IP address. This parameter is required if you select the Manual option.

  • Subnet Mask: Enter a subnet mask. This parameter is required if you select the Manual option.

  • Default Gateway: Enter the default gateway.

  • Primary DNS Server: Enter the IP address of the primary DNS server.

  • Secondary DNS Server: Enter the IP address of the secondary DNS server.

  • IP Binding: Select whether to bind the IP address to the virtual desktop by clicking the toggle button. This parameter is configurable if you select the Manual option.

  • Belong To: Select whether the virtual desktop belongs to a domain or workgroup. This parameter is configurable only when you use a Windows VM template.

  • Domain/Workgroup: Enter a domain name or workgroup name. This parameter is configurable only when you use a Windows VM template.

  • Login Account: Enter the account for logging in to the VM OS.

  • Login Password: Enter the password for logging in to the VM OS.

  • Confirm: Confirm the password.

  • Local Group: Select a local group for the login account. Options include Administrators, Power Users, and Users. This parameter is configurable only when you use a Windows VM template.

  • Activation Serial Number: Enter an activation serial number. This parameter is configurable only when you use a Windows VM template and select full initialization. If you do not set this parameter, the virtual desktop prompts for an activation serial number that matches the OS version after startup.
