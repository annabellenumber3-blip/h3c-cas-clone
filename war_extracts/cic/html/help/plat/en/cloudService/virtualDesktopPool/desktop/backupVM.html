Back up a virtual desktop immediately

Perform this task to back up a virtual desktop immediately to a local directory or a remote server.

Restrictions and guidelines

  • The system does not back up permission information about virtual desktops. After you restore a virtual desktop by using a backup file, you must reconfigure permission settings for the virtual desktop.

  • The system does not back up boot priority and automatic migration settings of virtual desktops. After you restore a virtual desktop by using a backup file, the boot priority and automatic migration settings of the virtual desktop are restored to the default. You can reconfigure these parameters by modifying the virtual desktop.

Procedure

  1. From the left navigation pane, select VDC > Virtual Desktop Pools > virtual desktop pool name.

  1. Click the Virtual Desktops tab.

  1. Right-click the target virtual desktop and then select Back Up Now from the shortcut menu.

  1. Specify the Backup File Name, Reserved Backups, Disk Read Rate Limit, Disk Write Rate Limit, Temporary Directory, Compression, and Backup Type parameters, and then click Next.

  1. Specify the backup location.

  • If you select the Host Local Directory option, enter a local directory path.

  • If you select the Remote Server option, set the required parameters and click Connectivity Test to verify that the specified server is reachable and the settings are correct.

  1. Select a backup mode. Options include Full, Incremental, and Differential.

  1. If you select Disk Backup, click Next to select the disks to be backed up.

  1. Click Finish.

Parameters

  • Backup File Name: Enter the name of the backup file.

  • Reserved Backups: Enter the number of reserved backups. If you do not configure this parameter, the system does not limit the number of backups.

  • Disk Read Rate Limit: Set the limit on the disk read rate during backup.

  • Disk Write Rate Limit: Set the limit on the disk write rate during backup.

  • Temporary Directory: Enter the temporary directory used for data restoration. The default is /vms/vmbackuptmp. If the directory you specify does not exist, the system automatically creates the directory during backup.

  • Compression: Set whether to compress the disk image during backup.

  • Backup Type: Select the mode of the backup.

  • VM Backup: Back up a virtual desktop as a whole, including all disks and configuration files of the virtual desktop. You can restore the virtual desktop by using the backup file.

  • Disk Backup: Back up one or multiple disks. You can restore only the disk data of the virtual desktop by using the backup file.

  • Backup Destination: Select the backup destination. If you select the Host Local Directory option, the system saves the backup to the local host. If you select the Remote Server option, the system saves the backup to a remote server.

  • Backup Location: Enter the directory for saving the backup file. If the directory you specify does not exist, the system automatically creates the directory during backup.

  • IP Address: Enter the IP address of the remote server, in dotted-decimal format.

  • Username: Enter the username for logging in to the remote server.

  • Password: Enter the password for logging in to the remote server.

  • FTP: Use FTP to transmit virtual desktop files.

  • SCP: Use SCP to transmit virtual desktop files.

  • Connectivity Test: Test the reachability of the remote server.

  • Backup Mode: Select the mode of the backup.

  • Full: Backs up all data and applications before a specific point of time for a virtual desktop.

  • Incremental: Backs up the data that has changed since the previous backup.

  • Differential: Backs up the data that has changed since the previous full backup.
