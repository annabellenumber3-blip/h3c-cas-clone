Migrate VMs between CVMs

Perform this task to manually migrate VMs between CVMs without service interruption.

Restrictions and guidelines

  • Do not connect hosts to a CVM during VM migration between the CVMs.

  • For successful migration of a VM the data cache mode of which is None or Directsync and the disk of which is in an NFS or AFS storage pool, shut down the VM before migration.

Procedure

  1. From the left navigation pane, select Cloud Services > Cloud Rainbow.

  1. Enter the name of the target VM and then press Enter, or double-click a host. All VMs on the host will be displayed.

  1. Drag the VM to the target host.

  1. In the dialog box that opens, enter the migration timeout, enable or disable compression, and then click Next.

  1. Select the target storage, where the disk of the VM is stored, and then click Finish.

Parameters

  • Migrate VM: Name of the VM.

  • Original Host: Name of the host where the VM to be migrated resides.

  • Target Host: Name of the host where the VM is to be migrated.

  • Timeout: Enter the migration timeout for migrating running VMs. For example, the migration timeout is set to 20 minutes. The system suspends a VM if the VM cannot be migrated within 20 minutes and restores the VM after the migration. If this parameter is set to 0, the system does not suspend a VM during migration.

  • Compress: Select this option to compress the data to be transmitted.
