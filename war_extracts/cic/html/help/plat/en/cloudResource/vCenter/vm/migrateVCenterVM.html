Migrating a VM in a VMware vCenter

Perform this task to migrate a VM in a VMware vCenter.

Restrictions and guidelines

  • When migrating an online VM, make sure the destination storage pool has sufficient storage resources and the destination host has sufficient CPU and memory resources. Resource insufficiency might cause migration failure, VM startup failure, or slow VM startup.

  • A VM cannot be migrated between an x86 host and an ARM host.

  • To avoid migration failure, do not shut down a VM during online migration.

Procedure

  1. From the left navigation pane, select Cloud Resources > VMware vCenter Name.

  1. Click the VM tab.

  1. Right-click the target VM, and then select Migrate.

  1. Configure the parameters as needed, and then click Finish.

Parameters

  • Migration Type: Select a migration type. Options include the following:

  • Host: Migrate the VM to another host without changing its storage pool. To use this type, make sure the VM uses a shared storage pool and the destination host is mounted to the shared storage pool.

  • Storage: Migrate the disks of the VM to another storage pool mounted to the VM's host, without changing the host.

  • Host and Storage: Migrate the VM to another host and the disks of the VM to a storage pool mounted to the destination host. The destination host can be any host in the vCenter.

  • Destination Storage Pool: Select a destination storage pool for the VM. If the VM has multiple disks, you can select different storage pools for the disks.
