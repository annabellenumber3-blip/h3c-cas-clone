Manage VMs

Unlike physical hosts, a VM is formed by a set of files instead of electronic components. Each VM is a complete system that contains the CPU, memory, network device, storage device, and BIOS. An OS or application runs on a VM in the same way it runs on a physical server. Compared with a physical host, a VM has the following advantages:

  • A VM has access to all resources of the host that accommodates it, such as CPUs, memory, disks, network devices, peripheral devices, and can run all applications.

  • VMs are isolated from one another by default, which provides secure data processing, network connection, and data storage.

  • Multiple VMs can reside on one host to make full use of hardware resources.

  • The image file and applications of a VM are compressed in files. VM deployment, backup, and restoration can be made through file replication.

  • A VM system, including the virtual hardware, OS, and applications, can flexibly migrate between hosts even when the VM is running.

  • VMs combine distributed resource management with high availability to provide applications with a service priority higher than the static physical architecture.

  • VMs can be created and distributed as plug-and-play virtual tools for fast deployment.

Features

  • Start VMs

  • Hibernate a VM

  • Restart a VM

  • Shut down a VM

  • Power off a VM

  • Delete a VM
