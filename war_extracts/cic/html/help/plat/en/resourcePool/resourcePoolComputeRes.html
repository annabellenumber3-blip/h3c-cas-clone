Display the compute resource of a resource pool

Perform this task to display compute resource information for a resource pool.

Restrictions and guidelines

The system does not display memory usage statistics if a resource pool uses a cloud resource of vCenter 6.0 or higher. vCenter 6.0 or higher does not support displaying memory usage graphs.

Procedure

  1. From the left navigation pane, select VDC > Resource Pools > Resource pool name.

  1. Click the Compute Resource tab.

Parameters

  • Resource Info: Brief information about the compute resource.

  • Compute Resource: Cloud resource cluster name.

  • CPUs: Number of CPUs in the resource pool.

  • CPU Overcommit: Proportion of the number of VM CPUs to the total number of CPUs in the resource pool.

  • Memory: Memory in the resource pool.

  • Memory Overcommit: Proportion of the VM memory to the total memory in the resource pool.

  • VMs: Number of VMs in the resource pool.

  • VM Distribution: Average number of VMs per host.

  • CPU Usage: CPU usage trend graph for the resource pool. The horizontal axis represents the time, and the vertical axis represents the average CPU usage.

  • Memory Usage: Memory usage trend graph for the resource pool. The horizontal axis represents the time, and the vertical axis represents the average memory usage.

  • VM State Statistics: Total number of VMs in the resource pool, and the number of VMs in each state.

  • CPU Usage (MHz): CPU frequency trend graph for the resource pool of the vCenter cloud resource. The horizontal axis represents the time, and the vertical axis represents the CPU frequency.

  • Memory Usage (MB): Memory usage trend graph for the resource pool of the vCenter cloud resource. The horizontal axis represents the time, and the vertical axis represents the memory usage.
