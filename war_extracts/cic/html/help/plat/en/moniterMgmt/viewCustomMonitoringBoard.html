Display monitoring information

Perform this task to display monitoring information through a customized monitoring panel.

Procedure

  1. From the left navigation pane, select Monitoring > Custom Monitoring.

  1. Click the target monitoring panel.

  1. To display monitoring information in full screen, click Full Screen. To exit full screen mode, press Esc.

Parameters

  • Health: Health scores of the current organization or resource pool (average heath scores of all hosts).

  • CPU Health:

0-50: The CPU usage is greater than 80%.

51-80: The CPU usage is smaller than 80% and greater than 60%.

81-100: The CPU usage is smaller than 60%.

  • Memory Health:

0-50: The memory usage is greater than 80%.

51-80: The memory usage is smaller than 80% and greater than 60%.

81-100: The memory usage is smaller than 60%.

  • Storage Health:

0-50: The storage usage is greater than 80%.

51-80: The storage usage is smaller than 80% and greater than 60%.

81-100: The storage usage is smaller than 60%.

  • Network Health:

0-50: The throughput is greater than 1024 Mbps.

51-80: The storage usage is smaller than 1024 Mbps and greater than 500 Mbps.

81-100: The storage usage is smaller than 500 Mbps.

  • Overall Health: CPU health × 30% + memory health × 40% + storage health × 15% + network health × 15%.

  • Host Health: Health scores of each host in the organization or resource pool.

  • CPU Overcommit:

  • Total Host CPUs: Total number of CPU cores for all hosts in the resource pool.

  • Total VM CPUs: Total number of CPU cores for all VMs in the resource pool.

  • CPU Overcommit: Percentage of total number of VM CPU cores to the total number of host CPU cores in the resource pool.

  • Memory Overcommit:

  • Total Host Memory: Total memory size of all hosts in the resource pool.

  • Total VM Memory: Total memory size of all VMs in the resource pool.

  • Memory Overcommit: Percentage of total VM memory size to the total host memory size in the resource pool.

  • Shared Storage Overcommit:

  • Total Storage: Total shared storage size in the current resource pool.

  • Available Storage: Available shared storage size in the current resource pool.

  • Storage Overcommit: Percentage of total used shared storage memory to the total shared storage memory size in the resource pool.

  • Host State Statistics: Total number of hosts and the numbers of hosts in normal, abnormal, and maintenance states in the current organization or resource pool.

  • VM State Statistics: Total number of VMs and the numbers of VMs in normal, shutdown, and abnormal states in the current organization or resource pool.

  • Top 5 Hosts by CPU Usage: Bar chart for the top 5 hosts with the highest CPU usage over the last hour in the current organization or resource pool. The horizontal axis represents the CPU usage, and the vertical axis represents the host name.

  • Top 5 Hosts by Memory Usage: Trend graph for the top 5 hosts with the highest memory usage over the last hour in the current organization or resource pool. The horizontal axis represents the time in a unit of 30 seconds, and the vertical axis represents the memory usage. To display the memory usage trend graph for a specific host, click the legend for the host.

  • Top 5 VMs by CPU Usage: Bar chart for the top 5 VMs with the highest CPU usage over the last hour in the current organization or resource pool. The horizontal axis represents the CPU usage, and the vertical axis represents the VM name.

  • Top 5 VMs by Memory Usage: Trend graph for the top 5 VMs with the highest memory usage over the last hour in the current organization or resource pool. The horizontal axis represents the time in a unit of 30 seconds, and the vertical axis represents the memory usage. To display the memory usage trend graph for a specific VM, click the legend for the VM.

  • Users: Total number of users and the numbers of users in valid and invalid states in the current organization.

  • CPU Statistics: CPU statistics for the current organization.

  • Quota: Maximum number of CPU cores that the organization can use.

  • Assigned: Total number of VM CPU cores assigned to the organization.

  • Overcommit: Ratio of the assigned CPU cores to the CPU core quota of the organization.

  • Memory Statistics: Memory statistics for the current organization.

  • Quota: Maximum memory size that the organization can use.

  • Assigned: Total memory size assigned to the VMs in the organization.

  • Overcommit: Ratio of the assigned memory size to the memory quota of the organization.

  • Storage Statistics: Storage statistics for the current organization.

  • Quota: Maximum storage size that the organization can use.

  • Assigned: Total storage size assigned to the VMs in the organization.

  • Overcommit: Ratio of the assigned storage size to the storage quota of the organization.

  • VMs: VM number information for the current organization.

  • Quota: Maximum number of VMs that can be created in the organization.

  • Assigned: Total number of VMs assigned to the organization.

  • Ratio: Ratio of the assigned VMs to the VM quota of the organization.
