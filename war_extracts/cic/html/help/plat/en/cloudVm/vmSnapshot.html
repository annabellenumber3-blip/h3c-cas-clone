Manage cloud host snapshots

Perform this task to create, restore, and delete cloud host snapshots.

Restrictions and guidelines

  • During snapshot creation or restoration, do not modify the configurations of the cloud host. Violation of this restriction might cause snapshot restoration failure.

  • Before you create a cloud host snapshot, make sure the storage that stores the cloud host image file has enough space. If the storage does not have enough space, snapshot creation fails, and the system displays the Failed to take snapshot...,maybe the storage is full or internal error Child process (/usr/bin/kvm-img snapshot -c sn-name /vms/target3/vm0.img status unexpected: exit status 1 message.

  • You can create snapshots only for cloud hosts that use intelligent disks.

Procedure

  1. From the left navigation pane, select Cloud Services > Cloud Hosts.

  1. Right-click the target cloud host and then select Snapshot Mgmt. from the shortcut menu, or select the target cloud host and then click Snapshot Mgmt..

  • To create a snapshot: Click Create. In the dialog box that opens, enter the snapshot name and description and click OK. If the cloud host is in running state, you can specify whether to include memory state in the snapshot and set the snapshot creation timeout.

  • To restore a cloud host with a snapshot: From the snapshot tree, select a snapshot and then click Restore. In the dialog box that opens, click OK.

  • To delete a snapshot: From the snapshot tree, select a snapshot and then click Delete. In the dialog box that opens, click OK.

Parameters

  • Memory Snapshot: Set whether to include memory state in the snapshot.

  • Timeout: Specify the timeout for taking the snapshot. If snapshot creation is not finished within this timeout, the system suspends the cloud host.
