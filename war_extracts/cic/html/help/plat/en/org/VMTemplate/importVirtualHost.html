Import VM configurations from a file

Perform this task to import configurations of multiple VMs from a file in bulk VM deployment of an organization.

Restrictions and guidelines

  • To avoid import failure, make sure the columns you specify match the columns in the file.

  • If an imported VM alias already exists in CIC, the VM cannot be deployed.

  • To avoid import failure, make sure the delimiter you select match the delimiters in the file.

Procedure

  1. From the left navigation pane, select VDC > Organizations > Organization name.

  1. Click the Templates tab.

  1. Click the  icon in the Action column for a VM template.

  1. Select a resource pool and set whether to enable quick deployment.

  1. Click Import from File.

  1. In the dialog box that opens, click Browse to select a file. You can click Preview to preview the columns in the file. Select a delimiter and specify the start row. Set whether to enable Configure OS.

  1. If you enable Configure OS, specify the initialization type and click Next.

  1. Specify the columns that indicate the alias, description, number of CPUs, number of CPU cores, memory size, and memory unit, respectively.

  1. Select the assignment mode.

  • If you select the Not Assign option, click Next.

  • If you select the Assign to User or Assign to User Group option, specify the column that indicates users or user groups, and click Next.

  • If you disable Configure OS, go to step 12. If you enable Configure OS, specify the columns that indicate the computer name and IP address assignment settings.

  • If you select the Manual option, specify the columns that indicate the IP address, subnet mask, default gateway, primary DNS server address, and secondary DNS server address, respectively. Select the Set IP Binding option as needed, and click Next.

  • If you select the DHCP option, click Next.

  1. Select whether the VMs belong to a domain or workgroup, and specify the columns that indicate the domain/workgroup, login account, password, and activation serial number, respectively. Select a local group.

  1. Click Finish.

Parameters

  • Select File: Select a file that contains VM configurations. You can create a file or click the Template File link to customize the template file. The system supports TXT or SCV files that use commas, semicolons, the Tab keys, or spaces to separate columns.

  • Delimiter: Select a column delimiter. Options include comma, semicolon, the tab key, and space.

  • Start Row: Specify the start row of configuration information.

  • Configure OS: Set whether to configure the OS for the VMs by clicking the toggle button.

  • Fast: Use CAStools to set OS parameters and user account information for the VMs and do not clear SID information of the OS. Fast initialization requires CAStools to be installed on the VM template. This mode applies to a Windows or Linux OS that supports CAStools.

  • Full: Use Sysprep to set OS parameters and user account information for the VMs and initialize SID information of the OS. Full initialization requires Sysprep to be installed on the VM template. This mode applies to Windows 7 and Windows XP.

  • CPUs: Select the column that indicates the number of CPUs. You can set this parameter only when the organization that the VMs belong to allows modification of VM configuration.

  • CPU Cores: Select the column that indicates the number of CPU cores. You can set this parameter only when the organization that the VMs belong to allows modification of VM configuration.

  • Memory Size: Set the memory size. You can set this parameter only when the organization that the VMs belong to allows modification of VM configuration.

  • Memory Unit: Select the unit of the memory. You can set this parameter only when the organization that the VMs belong to allows modification of VM configuration.

  • Assignment Mode: Select the objects you want to assign the VMs to.

  • User/User Group: Select the column that indicates the users or user groups. You must set this parameter if you select the Assign to User or Assign to User Group option.

  • Computer Name: Select the column that indicates the computer name.

  • IP Assignment: Select an IP address assignment mode. Options include DHCP and Manual.

  • IP Address: Select the column that indicates the IP address. This parameter is required if you select the Manual option.

  • Subnet Mask: Select the column that indicates the subnet mask. This parameter is required if you select the Manual option.

  • Default Gateway: Select the column that indicates the default gateway.

  • Primary DNS Server: Select the column that indicates the primary DNS server address.

  • Secondary DNS Server: Select the column that indicates the secondary DNS server address.

  • IP Binding: Select whether to bind the IP addresses to the VMs by clicking the toggle button. This parameter is configurable if you select the Manual option.

  • Belong To: Select whether the virtual desktops belong to a domain or workgroup. This parameter is configurable only when you use a Windows VM template.

  • Domain/Workgroup: Select the column that indicates the domain or workgroup. This parameter is configurable only when you use a Windows VM template.

  • Login Account: Select the column that indicates the login account.

  • Password: Select the column that indicates the password.

  • Local Group: Select a local group for the login account. Options include Administrators, Power Users, and Users. This parameter is configurable only when you use a Windows VM template.

  • Activation Serial Number: Enter the activation serial number. This parameter is configurable only when you use a Windows VM template and select full initialization. If you do not set this parameter, a VM prompts for an activation serial number that matches the OS version after startup.
