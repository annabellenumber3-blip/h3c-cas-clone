Deploy a VM for an organization in a vCenter cloud resource

Perform this task to deploy a VM by using a VM template for an organization in a vCenter cloud resource. You can specify the VM assignment mode when deploying the VM.

Procedure

  1. From the left navigation pane, select VDC > Organizations > Organization name.

  1. Click the Templates tab.

  1. Click the  icon in the Action column for a VM template.

  1. Enter the VM alias and description, select a cluster, select whether to configure OS, and click Next. If the organization that the VM belongs to allows modification of VM configuration, configure CPU and memory settings as described in step 5. If the organization that the VM belongs to forbid modification of VM configuration, go to step 6.

  1. Specify the number of CPUs, the number of CPU cores, and the memory size.

  1. Select the assignment mode.

  • If you select the Assign to User option, click the  icon for the User field. In the dialog box that opens, select users from the Users Not Assigned the VM list and click Add. Click OK.

  • If you select Assign to User Group, click the  icon for the User Group field. In the dialog box that opens, select user groups and click OK.

  • If you select Not Assign, the VM is not assigned to anyone after it is created.

  1. If you have enabled Configure OS, click Next. If you have disabled Configure OS, click Finish.

  1. Enter the computer name and select an IP address assignment mode.

  • If you select the DHCP option, click Next.

  • If you select the Manual option, enter the IP address, subnet mask, default gateway, primary DNS server address, and secondary DNS server address, and click Next.

  1. Configure login information.

  • If you use a Windows VM template, specify a domain or workgroup, enter the login account and password, enter the activation serial number, and select a local group and time zone.

  • If you use a Linux VM template, enter the login account and password.

  1. Click Finish.

Parameters

  • Resource Pool: Select the resource pool used to deploy the VM.

  • Configure OS: Set whether to configure the OS for the VM by clicking the toggle button.

  • If you enable this setting for Windows, Sysprep is used to set OS parameters and user account information for the VM and clear SID information of the OS. This setting requires Sysprep to be installed on the VM template.

  • If you enable this setting for Linux, you must configure Perl Scripts and VMware Tools on the VM template and make sure the OS is installed on disk scsi0:0.

  • Assignment Mode: Select the objects you want to assign the VM to.

  • Not Assign: Do not assign the VM.

  • Assign to User: Assign the VM to users.

  • Assign to User Group: Assign the VM to user groups.

  • User: Select the users to be assigned the VM. This parameter is required if you select the Assign to User option. Click the  icon to the right of this parameter to select one or multiple users.

  • User Group: Select the user groups to be assigned the VM. This parameter is required if you select the Assign to User Group option. Click the  icon to the right of this parameter to select one or multiple user groups.

  • Computer Name: Enter a computer name.

  • IP Assignment: Select an IP address assignment mode.

  • IP Address: Enter an IP address.

  • Subnet Mask: Enter a subnet mask.

  • Default Gateway: Enter the default gateway.

  • Primary DNS Server: Enter the IP address of the primary DNS server.

  • Secondary DNS Server: Enter the IP address of the secondary DNS server.

  • Belong To: Select whether the VM belongs to a domain or workgroup. This parameter is configurable only when you use a Windows VM template.

  • Domain/Workgroup: Enter a domain name or workgroup name. This parameter is configurable only when you use a Windows VM template.

  • Login Account: Enter the account for logging in to the VM OS.

  • Password: Enter the password for logging in to the VM OS.

  • Confirm: Confirm the password.

  • Local Group: Select a local group for the login account. Options include Administrators, Power Users, and Users. This parameter is configurable only when you use a Windows VM template.

  • Activation Serial Number: Enter the activation serial number. This parameter is configurable only when you use a Windows VM template and select Configure OS. If you do not set this parameter, the VM prompts for an activation serial number that matches the OS version after startup.
