Manage VM snapshots

Perform this task to create, restore, and delete VM snapshots.

Restrictions and guidelines

  • During snapshot creation or restoration, do not modify the configurations of the VM. Violation of this restriction might cause snapshot restoration failure.

  • Before you create a VM snapshot, make sure the storage that stores the VM image file has enough space. If the storage does not have enough space, snapshot creation fails, and the system displays the Failed to take snapshot...,maybe the storage is full or internal error Child process (/usr/bin/kvm-img snapshot -c sn-name /vms/target3/vm0.img status unexpected: exit status 1 message.

  • You can create snapshots only for VMs that use intelligent disks.

Procedure

  1. From the left navigation pane, select VDC > Organizations > Organization name.

  1. Click the VMs tab.

  1. Right-click the target VM and then select Snapshot Mgmt. from the shortcut menu.

  • Create a snapshot by using the following procedure: Click Create. In the dialog box that opens, enter the snapshot name and description and then click OK. If the VM is in running state, you can specify whether to include memory state in the snapshot and set the snapshot creation timeout.

  • Restore a VM with a snapshot by using the following procedure: From the snapshot tree, select a snapshot and then click Restore. In the dialog box that opens, click OK.

  • Delete a snapshot by using the following procedure: From the snapshot tree, select a snapshot and then click Delete. In the dialog box that opens, click OK.

Parameters

  • Memory Snapshot: Set whether to include memory state in the snapshot.

  • Timeout: Specify the timeout for taking the snapshot. If snapshot creation is not finished within this timeout, the system suspends the VM.
