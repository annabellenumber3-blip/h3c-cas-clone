Dashboard

The CIC dashboard displays the numbers of pending workflows, the number of pending support tickets, organization resource statistics, and resource pool resource statistics. Organization resource statistics include VM state statistics, user statistics, CPU and memory statistics, and storage and VM statistics. Resource pool resource statistics include the health index, CPU overcommitment, memory overcommitment, and shared storage overcommitment.

Restrictions and guidelines

Information on the dashboard is refreshed every 10 minutes. You can manually synchronize information in each vCenter resource to refresh the dashboard.

Procedure

  1. Log in to CIC.

  1. To delegate the approval authority over workflows, click Approval Authority Delegation. You can delegate all workflows or delegate workflows by workflow types.

  1. To view the workflows of the corresponding type, click a workflow icon.

  1. To view the summary information for an organization, double-click an organization icon.

Parameters

  • Pending Workflow: Numbers of pending workflows, including cloud host workflows, cloud disk workflows, user preregistration workflows, and cloud backup policy workflows.

  • Pending Support Ticket: Number of pending support tickets.

  • Organization Resource Statistics: Resource statistics about an organization, including VM state statistics, user statistics, CPU and memory statistics, and storage and VM statistics.

  • VM State Statistics: Total number of VMs in the organization, and the number of VMs in each state.

  • Users: Total number of users in the organization, and the number of users in each state.

  • CPU/Memory Statistics: CPU and memory statistics.

  • Quota: Maximum number of CPUs that can be assigned to the organization.

  • Assigned: Current number of CPUs assigned to the VMs of the organization.

  • Ratio: Number of assigned CPUs divided by CPU quota.

  • Quota: Maximum memory size that can be assigned to the organization.

  • Assigned: Current memory size assigned to the VMs of the organization.

  • Ratio: Assigned memory size divided by memory quota.

  • Storage/VM Statistics: Storage and VM statistics.

  • Quota: Maximum storage size that can be assigned to the organization.

  • Assigned: Current storage size assigned to the VMs of the organization.

  • Ratio: Assigned storage size divided by storage quota.

  • Quota: Maximum number of VMs that the organization can manage.

  • Assigned: Current number of VMs assigned to the organization.

  • Overcommitment: Number of assigned VMs divided by VM quota.

  • Resource Statistics of Resource Pool

  • Health

  • CPU health index: The CPU health index value range is as follows: 0 to 50 when the host CPU usage is higher than 80%. 51 to 80 when the host CPU usage is in the range of 60% to 80%. 81 to 100 when the host CPU usage is lower than 60%.

  • Memory health index: The memory health index value range is as follows: 0 to 50 when the host memory usage is higher than 80%. 51 to 80 when the host memory usage is in the range of 60% to 80%. 81 to 100 when the host memory usage is lower than 60%.

  • Storage health index: The storage health index value range is as follows: 0 to 50 when the host storage usage is higher than 80%. 51 to 80 when the host storage usage is in the range of 60% to 80%. 81 to 100 when the host storage usage is lower than 60%.

  • Network health index: The network health index value range is as follows: 0 to 50 when the network throughput is higher than 1024 Mbps. 51 to 80 when the network throughput is in the range of 500 to 1024 Mbps. 81 to 100 when the network throughput is lower than 500 Mbps.

  • Overall health index: Overall health index=CPU health index × 30% + memory health index × 40% + storage health index × 15% + network health index × 15%

  • Host health index: Health index of a host in the resource pool.

  • CPU Overcommit

  • Total Host CPUs: Total number of host CPU cores in the resource pool.

  • Total VM CPUs: Total number of VM CPU cores in the resource pool.

  • CPU Overcommit: Host CPU usage, which is the proportion of the total number of VM CPU cores to the total number of host CPU cores.

  • Memory Overcommit

  • Total Host Memory: Total host memory size of the resource pool.

  • Total VM Memory: Total VM memory size of the resource pool.

  • Memory Overcommit: Host memory usage, which is the proportion of the total VM memory to the total host memory.

  • Shared Storage Overcommit

  • Total Storage: Total shared storage size of the resource pool.

  • Available Storage: Available shared storage size of the resource pool.

  • Storage Overcommit: Shared storage usage, which is the proportion of the available shared storage size to the total shared storage size.
